# しりとり

Webブラウザで遊べるしりとりアプリです。CPUとの対戦と、友達とのオンライン対戦ができます。

jig.jpサマーインターンシップ2026 Webコース選考課題として制作しました。

## デプロイ先URL

https://shiritori.miyamegmilk.deno.net/

## 遊び方

### CPUと対戦
ランダムな単語からスタートし、コンピュータと交互に単語を出し合います。
コンピュータは約15万語の辞書から「続けられる単語」を探して返してきます。
続く単語をコンピュータが思いつけなければあなたの勝ちです。

### オンライン対戦
2人でリアルタイムに対戦できます。マッチング方法は2種類あります。

- **さっと対戦** — 待っている人とランダムにマッチング（制限時間30秒）
- **合言葉で対戦** — 部屋を作って合言葉（または共有URL）を友達に伝えて対戦

部屋を作るときは、部屋主が以下のルールを自由に設定できます。

| 設定 | 選択肢 |
|------|--------|
| 制限時間 | なし / 10秒 / 20秒 / 30秒 / 60秒（時間切れは負け） |
| 先手 | ランダム / 自分 / 相手 |
| 辞書判定 | あり（実在する単語のみ） / なし（なんでもOK） |

## ルール

- ひらがな2文字以上で入力します（長音「ー」も使えます）
- 前の単語の末尾の文字から始まる単語だけが有効です
  - 拗音・促音は大きい文字として扱います（「きしゃ」→「や」から）
  - 末尾の長音は直前の文字で継ぎます（「こーひー」→「ひ」から）
- 「ん」で終わる単語を出すと負けです
- すでに出た単語をもう一度出すと負けです
- ゲーム中・ゲーム終了後いつでもリセットできます（CPU対戦）

## 実装した機能

- **必須仕様**: 直前の単語の表示 / 単語の入力 / 接続判定とエラー表示 / 「ん」終わりでゲーム終了 / 既出単語でゲーム終了 / リセット
- **辞書判定**: 約75万語のひらがな名詞リストで、実在しない単語をはじきます（詳細は「辞書データについて」）
- **CPU対戦**: サーバーが辞書から次の一手を探して返答します。「ん」で終わらない単語を優先し、それしか無ければ「ん」を出して負けます
- **オンライン対戦**: WebSocketによるターン制の2人対戦。ランダムマッチと合言葉ルームの2方式
- **部屋設定**: 制限時間・先手・辞書判定の有無を部屋主がカスタマイズ可能
- **スコア表示**: 出した単語の文字数を得点として集計・表示
- **効果音**: Web Audio APIで生成（正解・エラー・勝敗・残り5秒のカウント音）。ミュート切替つき
- **アニメーション**: 単語のポップ、エラー時のシェイク、履歴のスライドイン、勝敗表示のバウンド

## デザイン

シンプルで見やすいカードベースのデザインにしました。
今の単語を画面中央に大きく表示して「何に続ければいいか」が一目で分かるようにし、
誰がどの単語を出したか・何点入ったかを履歴にタグで添えています。
スマートフォンでも遊べるよう1カラムのレイアウトにしています。

## 技術構成

- **ランタイム**: Deno（フレームワークなし）
- **フロントエンド**: HTML / CSS / vanilla JavaScript
- **リアルタイム通信**: WebSocket（Deno標準）
- **デプロイ**: Deno Deploy

### 構成のポイント

- CPU対戦はステートレスなREST API（履歴はクライアントが保持し、サーバーは検証とCPUの返答だけを担当）。
  サーバーが複数インスタンスにスケールしても状態の不整合が起きません
- オンライン対戦の勝敗・タイマー・スコアはすべてサーバー側で判定し、クライアントは表示に専念
- 辞書は起動時に読み込み、先頭文字で索引化してCPUの手探しを高速化

## 辞書データについて

辞書は2層構成です。どちらもひらがな2〜15文字の名詞の読みだけを対象にしています。

- `dict/words.txt`（約15万語） — MeCab用の形態素解析辞書 IPAdic の名詞
  （一般名詞＋地名・人名などの固有名詞）から抽出（`scripts/build_dict.py`）。
  **入力の受理判定とCPUの語彙**の両方に使います
- `dict/words_extra.txt`（約60万語） — 新語辞書 mecab-ipadic-NEologd の種データから、
  表層形がカナ・英数字の語（「スマホ」「アプリ」などのカタカナ語・新語）を
  抽出（`scripts/build_extra_dict.py`）。**入力の受理判定にだけ**使います。
  NEologdには俗語なども含まれるため、CPUの返答には使わない設計です

しりとりのルールとしては伝統的な「名詞のみ」で、動詞・形容詞は使えません。

## AIの活用について

本アプリは Claude Code（Anthropic）を全面的に活用して開発しました。役割分担は次の通りです。

- **自分が担当したこと**: 作る機能の選定と仕様の決定（一人用モードのCPU対戦への置き換え、部屋設定に入れるルール項目、スコアのルールなど）、実際に遊んでの受け入れ確認、公開してよいかの最終判断
- **Claude Codeに任せたこと**: 設計の詳細化（ステートレスなCPU対戦API、WebSocketのメッセージ設計など）、コーディング、PlaywrightとfetchによるAPI/UI自動テストの作成・実行
- **進め方**: 「1機能ずつ実装 → 自動テストで全ケース検証 → コミット」の繰り返しで、各機能の動作を確認してから次の機能に進めています

辞書データの抽出スクリプト（IPAdicからの変換）もClaude Codeで作成しています。
